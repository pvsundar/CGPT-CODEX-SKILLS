## Copy-Ready LinkedIn Post

I have started building a public repository of Codex skills for academic,
teaching, manuscript, and research workflows.

One concrete example is this interactive flipcard deck for scholars
learning what GitHub repositories are and why public versioned sharing
matters:

<https://htmlpreview.github.io/?https://github.com/pvsundar/CGPT-CODEX-SKILLS/blob/main/cgpt-canvas-flipcards/examples/study-flipcards-github-repos-for-scholars.html>

The motivation is practical. Many of us are figuring out useful
AI-assisted workflows for academic work, but those workflows often stay
buried in one person's local setup or chat history. A Codex skill turns a
repeatable workflow into a small, inspectable instruction set that others
can run, adapt, critique, and improve.

I built the repository to make some of these workflows more explicit and
shareable.

It currently includes skills for:

- academic DOCX authoring and journal-ready Word formatting
- UW and UW Bothell slide deck creation
- Quarto render checking and debugging
- manuscript recovery, handoffs, and thread closeout
- citation, statistics, and data visualization audits
- Canvas teaching tools
- safer Git workflows in synced folders

Repository: <https://github.com/pvsundar/CGPT-CODEX-SKILLS>

The flipcards example is intentionally small, but it shows the broader
value. A well-scoped skill can help Codex generate a polished,
self-contained teaching artifact that works locally, can be hosted online,
and can be embedded in Canvas.

If you are a scholar, teacher, instructional designer, librarian, doctoral
student, research software engineer, or simply someone building repeatable
academic workflows, I would welcome your use, critique, and contributions.

You can help by:

- using the existing skills and telling me what works or breaks
- improving the instructions so they are clearer and safer
- adding new Codex skills for scholarly workflows
- contributing concrete examples from teaching, writing, analysis, or
  project management
- testing these in real classrooms, labs, and research projects
- helping make AI-assisted academic work more transparent, inspectable,
  and reusable

This is not a finished product. It is a public starting point. If you have
a workflow you repeat often, there is a good chance it could become a
useful skill for others.

Please use the repo, fork it, open issues, suggest skills, or contribute
examples from your own scholarly workflows.
